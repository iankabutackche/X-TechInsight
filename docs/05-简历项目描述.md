## 三、精简版 Bullet（推荐，3～4 条）

适合简历空间有限：

- 设计并实现 **RAG 文档问答链路**：PDF/MD/TXT 上传 → 文本切分 → OpenAI Embedding → ChromaDB 向量检索 → GPT 流式生成，资料不足时 **拒答** 降低幻觉  
- 实现 **多对话空间 + 领域标签（domain）过滤**，检索阶段按 tag 缩小向量搜索范围，前端支持按对话挂载文档与冲突提醒  
- 基于 **FastAPI + SQLite** 构建 REST/SSE API（对话 CRUD、异步上传入库、流式问答），React 三栏 UI + Markdown 代码高亮渲染  
- 完成 **Render 前后端分离部署**（Static Site + Web Service），处理 CORS、构建时环境变量、免费 tier 休眠等工程问题

## 四、完整版 Bullet（5～6 条，技术岗详细版）

- 独立开发 **X-TechInsight** 全栈 RAG 平台：用户上传技术文档，系统切分向量化后支持自然语言问答，回答附带 `[Source N]` **可溯源引用**  
- 使用 **LangChain + ChromaDB** 实现向量入库与 Top-K 语义检索；通过 metadata `domain` 字段实现 **领域隔离**，避免跨领域文档干扰检索结果  
- 在 **agent_logic** 层设计 System Prompt 约束：仅依据检索片段作答、无法回答时明确拒答，平衡可用性与可信度  
- 后端 **FastAPI**：`BackgroundTasks` 异步解析大文件；`/query/stream` 基于 **SSE** 推送 token/sources/done 事件；SQLAlchemy 管理 Chats/Messages 持久化  
- 前端 **React + Vite**：三栏交互（对话列表 / 聊天 / 配置面板），流式消息渲染、`MessageMarkdown` 代码高亮，生产环境 `VITE_API_BASE_URL` 配置  
- **云部署与排错**：Render 部署前后端分离服务，配置 CORS/OpenAI Key/构建环境变量；编写部署与排错文档，公网 Demo 可演示

## 六、项目描述段落版（部分网站要段落而非 bullet）

```
X-TechInsight 是我独立开发的模块化 RAG 技术文档分析平台。用户可上传 PDF、Markdown
或纯文本，系统自动切分并向量化存入 ChromaDB；提问时按领域标签检索 Top-K 相关片段，
由 GPT 基于片段流式生成回答，并标注可溯源来源。后端采用 FastAPI + SQLite，前端为
React 三栏界面，支持多对话并存、文档挂载与领域冲突提醒。项目已部署至 Render 公网，
完整覆盖从文档入库、检索增强生成到前后端联调与 CORS/环境变量等工程化问题。
```
