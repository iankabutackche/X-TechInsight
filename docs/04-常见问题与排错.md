## 二、公网部署（Render）问题

### 2.1 Failed to fetch

**现象：** 弹窗或 Console 显示 `Failed to fetch`；无法新建对话、无法加载列表。

**常见原因：**

| 原因 | 说明 |
|------|------|
| 后端休眠 | Render 免费版 15 分钟无访问会睡眠，唤醒需 30～60 秒 |
| CORS 未配置 | 浏览器拦截跨域请求（见 §2.2） |
| API 地址错误 | 前端请求了错误的后端 URL（见 §2.3） |
| 后端崩溃 | OpenAI Key 未配、依赖缺失等 |

**解决步骤：**

1. 浏览器打开 https://x-techinsight.onrender.com/health  
   - 若很久才返回 `{"status":"ok"}` → 正常，是 **休眠唤醒**  
   - 若一直打不开 → 去 Render 后端 Dashboard 看 **Logs**
2. `Cmd + Shift + R` 强制刷新前端
3. 再点「新建对话」重试
4. 仍失败 → 按 §2.2、§2.3 检查环境变量并 **重新 Deploy**

---

### 2.2 CORS 跨域报错

**现象（Console 红色）：**

```text
Access to fetch at 'https://x-techinsight.onrender.com/...' from origin
'https://x-techinsight-web.onrender.com' has been blocked by CORS policy:
No 'Access-Control-Allow-Origin' header is present...
```

**原因：** 后端没有允许前端域名跨域访问。

**解决步骤：**

1. 登录 Render → 打开 **后端** Web Service（`x-techinsight`）
2. **Environment** 添加或修改：

   | Key | Value |
   |-----|--------|
   | `CORS_ORIGINS` | `https://x-techinsight-web.onrender.com` |

   > 多个域名用英文逗号分隔，**不要** 加末尾斜杠。

3. **Save Changes** → 等待自动 Redeploy
4. 验证（Mac 终端）：

   ```bash
   curl -s -D - -o /dev/null \
     -H "Origin: https://x-techinsight-web.onrender.com" \
     https://x-techinsight.onrender.com/health | grep -i access-control
   ```

   应看到：`access-control-allow-origin: https://x-techinsight-web.onrender.com`

**代码兜底：** 项目 `backend/main.py` 已加 `allow_origin_regex` 匹配 `*.onrender.com`；若 GitHub 已 push 最新代码，即使忘记配环境变量也可能生效。

---

### 2.3 环境变量未生效

**现象：** Console 里请求 URL 是：

```text
VITE_API_BASE_URL/chats/list
```

**原因：** Vite 在 **`npm run build` 时** 把 `VITE_API_BASE_URL` 写进 JS。构建时没设或设错，就会留下字面量。

**解决步骤：**

1. Render → **前端** Static Site → **Environment**
2. 确认：

   | Key | Value |
   |-----|--------|
   | `VITE_API_BASE_URL` | `https://x-techinsight.onrender.com` |

   ❌ 错误示例：`VITE_API_BASE_URL`（把变量名当值）  
   ❌ 错误示例：`https://x-techinsight.onrender.com/`（末尾多斜杠一般可接受，但建议无斜杠）

3. **Manual Deploy → Clear build cache & deploy**（必须清缓存重建）
4. 代码兜底：`frontend/src/api.js` 在 `onrender.com` 会自动 fallback 到后端地址

---

### 2.4 前端根路径 404

**现象：**

- `https://x-techinsight-web.onrender.com/` → 404  
- `https://x-techinsight-web.onrender.com/index.html` → 正常

**原因：** React 是单页应用（SPA），静态托管默认没有 `/` → `index.html` 的重写规则。

**临时办法：** 书签使用带 `index.html` 的完整地址。

**永久办法（待做）：** 在 `frontend/public/_redirects` 添加：

```text
/*    /index.html   200
```

然后 push 并重新 Deploy 前端。

## 三、使用层面的「看起来像报错」

### 3.1 「拒答」不是系统坏了

**现象：** AI 回答：

```text
根据现有资料，我无法回答这个问题。
```

**原因：** RAG 设计如此——检索不到足够相关的文档片段时 **故意拒答**，避免瞎编。

**排查清单：**

| 检查项 | 处理 |
|--------|------|
| 是否已上传文档 | 右侧先上传 `.pdf/.md/.txt` |
| 是否等够解析时间 | 上传后等 10～30 秒再提问 |
| 问题是否与文档相关 | 换文档里能直接找到答案的问题 |
| 领域 tag 是否一致 | 对话 tag 需与文档入库时的 tag 一致 |
| 公网是否丢数据 | Render 免费版重启可能清空库，需重新上传 |

---

### 3.2 领域冲突提醒

**现象：** 右侧红色框「领域冲突提醒」。

**原因：** 你 **勾选挂载** 的文档中，有文档 tag 与当前对话 tag 不一致。

**处理：** 取消勾选不匹配的文档 / 改对话 tag / 新建对应对话。  
这不是程序崩溃，是 **UI 警告**。

## 四、OpenAI / AI 相关

### 4.1 OpenAI 连接失败（国内）

**现象（后端终端 / Render Logs）：**

- `Connection error`
- `SSL: CERTIFICATE_VERIFY_FAILED`
- `APIConnectionError`

**原因：** 国内网络访问 OpenAI 不稳定；本地未开代理。

**本地解决（Mac）：**

```bash
# 假设 Clash 监听 7890，按你的实际端口改
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890

cd ~/Desktop/X-TechInsight/backend
source venv/bin/activate
python main.py
```

**Render 上：** 服务器在海外，一般 **不需要** 代理；只需配好 `OPENAI_API_KEY`。

**检查 Key：**

```bash
# 本地 backend/.env 中应有（勿提交 Git）
OPENAI_API_KEY=sk-...
```

---

### 4.2 OpenAI 429 或配额

**现象：**

```text
Error code: 429
Rate limit exceeded
Insufficient quota
```

**原因：** API 调用超限或账户 **未充值 / 余额用尽**。

**解决：**

1. 登录 https://platform.openai.com → **Billing** 查看余额
2. 充值或绑定支付方式
3. 免费额度用完后必须付费才能继续调用
4. 重试前等 1～2 分钟（若是短时 rate limit）

## 五、本地开发问题

### 5.1 前端 5173 无法访问

**现象：** 浏览器打不开 `http://localhost:5173`

**排查：**

```bash
cd ~/Desktop/X-TechInsight/frontend
npm install
npm run dev
```

终端应显示：

```text
Local: http://localhost:5173/
```

| 情况 | 处理 |
|------|------|
| 端口被占用 | 关掉占用 5173 的程序，或改 `vite.config.js` 里 `server.port` |
| `npm` 报错 | 确认已安装 Node.js（建议 18+） |
| 页面白屏 | 打开浏览器 Console 看具体 JS 报错 |

**本地 API 不走环境变量：** `api.js` 在 localhost 时 `API_BASE` 为空，请求走 Vite 代理（见 `vite.config.js`）。

---

### 5.2 后端 8000 无法访问

**现象：** 前端能开，但所有请求失败；或 `http://127.0.0.1:8000/health` 打不开。

**排查：**

```bash
cd ~/Desktop/X-TechInsight/backend
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

| 情况 | 处理 |
|------|------|
| `OPENAI_API_KEY` 未配置 | 创建 `backend/.env` 并写入 Key |
| 8000 被占用 | 换端口或结束占用进程 |
| 模块找不到 | 确认 venv 已 activate |

验证：

```bash
curl http://127.0.0.1:8000/health
# 期望：{"status":"ok"}
```

---

### 5.3 PDF 相关

| 报错 / 现象 | 原因 | 处理 |
|-------------|------|------|
| 加密 PDF | 文件有密码保护 | 解密后重传，或导出为 `.md`/`.txt` |
| 扫描版 PDF | 无法选中文字，提取为空 | OCR 后重传，或改用 Markdown |
| 文件超过 10MB | 超出 `main.py` 限制 | 压缩或拆分文档 |
| 暂不支持的格式 | 非 pdf/md/txt | 转换格式后上传 |

## 六、Git 与 Render 部署

### 6.1 git push 认证失败

**现象：**

```text
remote: Permission to xxx/X-TechInsight.git denied
# 或提示用户名是 iankabutack 而非 iankabutackche
```

**原因：** GitHub 用户名 / 凭据与仓库所有者不一致。

**处理：**

1. 确认仓库地址：`git@github.com:iankabutackche/X-TechInsight.git`
2. 使用 Personal Access Token 或 SSH Key 登录 **正确账号**
3. 推送：

   ```bash
   cd ~/Desktop/X-TechInsight
   git push origin main
   ```

---

### 6.2 Render 部署 GitHub

**现象：** Private 仓库连接失败、Deployment Credentials 为空、循环授权。

**已验证可行的做法：**

| 方案 | 说明 |
|------|------|
| 改为 Public 仓库 | 最简单，Render 直接连 |
| Public Git Repository URL | 手动填仓库 HTTPS 地址 |
| 检查 GitHub 授权 | Render Dashboard → Account Settings → GitHub 重新连接 |

**后端 Render 配置参考：**

| 项 | 值 |
|----|-----|
| Root Directory | `backend` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `uvicorn main:app --host 0.0.0.0 --port $PORT` |
| `OPENAI_API_KEY` | 在 Environment 配置 |
| `PYTHON_VERSION` | `3.11.0` |

**前端 Static Site：**

| 项 | 值 |
|----|-----|
| Root Directory | `frontend` |
| Build Command | `npm install && npm run build` |
| Publish Directory | `dist` |
| `VITE_API_BASE_URL` | `https://x-techinsight.onrender.com` |

---

### 6.3 Render 免费版数据丢失

**现象：** 昨天上传的文档、对话历史今天没了。

**原因：** 免费 Web Service 的 **本地磁盘不持久**；重启、重新部署、休眠唤醒后 SQLite / ChromaDB 可能清空。

**应对：**

| 场景 | 建议 |
|------|------|
| 面试演示 | 演示前 **重新上传** 文档，演示前 5 分钟先访问一次唤醒后端 |
| 长期使用 | 换付费 Persistent Disk，或 PostgreSQL + 云向量库 |

---

### 6.4 Render 休眠唤醒

**现象：** 第一次打开网页很慢；第一次请求失败，第二次成功。

**原因：** 免费档自动休眠，冷启动约 30～60 秒。

**演示技巧：**

1. 录屏前 2 分钟先打开 `/health` 唤醒后端  
2. 口播时可说：「免费云托管会休眠，生产环境会用付费实例或保活策略」  
3. 这本身可作为 **面试加分点**（你理解 serverless/免费 tier 限制）

## 八、排错通用流程

遇到任何问题时，按此顺序：

```
1. 浏览器 Console（F12）→ 看红色报错原文
2. 后端是否存活 → /health
3. 请求 URL 是否正确 → 不应出现 VITE_API_BASE_URL 字面量
4. CORS 头是否存在 → curl + Origin 头测试
5. Render Logs → 后端崩溃 / OpenAI 报错
6. 本地则查 .env、代理、端口
```

## 十、仍无法解决时

收集以下信息（截图或复制文字）：

1. 你在 **本地** 还是 **公网** 遇到问题  
2. Console **完整红色报错**（Network 里失败请求的 URL 和 Status）  
3. `https://x-techinsight.onrender.com/health` 的返回结果  
4. Render 后端 Logs 最后 20 行  

便于精准定位，而不是反复「Failed to fetch」。
