# X-TechInsight — 录屏演示脚本

> 按本脚本 **3～5 分钟** 录一条完整演示视频，可用于作品集、面试前发送。  
> Mac 录屏：**Cmd + Shift + 5** → 选「录制所选部分」或全屏。

---

## 一、录屏前准备（必做）

### 1.1 环境

| 项 | 要求 |
|----|------|
| 浏览器 | Chrome / Safari，**无痕窗口** 或清缓存，避免旧报错 |
| 网址 | https://x-techinsight-web.onrender.com/index.html |
| 唤醒后端 | 录屏 **2 分钟前** 打开 https://x-techinsight.onrender.com/health ，看到 `{"status":"ok"}` |
| 演示文档 | 准备 1 个 **`.md` 文件**（建议：项目 README 复制一份，或自建 `demo-tech-stack.md`） |

### 1.2 演示文档内容建议（`demo-tech-stack.md`）

若单独建文件，可包含：

```markdown
# X-TechInsight 技术栈

- 前端：React + Vite
- 后端：FastAPI + SQLite
- 向量库：ChromaDB
- AI：LangChain + OpenAI（Embedding + gpt-4o-mini）
- 部署：Render 前后端分离
```

### 1.3 录屏设置

- 分辨率：1920×1080 或 1440×900  
- 关闭无关通知（Focus / 勿扰）  
- 浏览器只留 **一个标签页**  
- 可选：开麦克风录旁白（下文有旁白稿）

## 二、分镜脚本（总时长约 4 分钟）

### 镜头 1 · 开场（0:00～0:25）

**画面：** 浏览器打开前端首页，中间显示「X-TechInsight」和「点击左侧新建对话开始」。

**旁白：**

> 这是 X-TechInsight，一个基于 RAG 的技术文档问答平台。  
> 用户可以上传文档，系统检索相关片段后再让大模型回答，并标注来源，减少幻觉。

**操作：** 无，停留 2 秒展示界面三栏布局。

---

### 镜头 2 · 新建对话（0:25～0:50）

**画面：** 点击左侧 **「+ 新建对话」**。

**旁白：**

> 我先创建一个对话，名称叫「技术栈讨论」，领域标签选「后端」。  
> 这个 tag 会在向量检索时过滤，只查同领域的文档片段。

**操作：**

1. 对话名称：`技术栈讨论`  
2. 领域标签：`后端`  
3. 点击创建  

**检查点：** 左侧出现新对话，中间可输入。

---

### 镜头 3 · 上传文档（0:50～1:30）

**画面：** 右侧配置面板。

**旁白：**

> 接下来上传一份 Markdown 文档。上传后系统会在后台切分、向量化，写入 ChromaDB。

**操作：**

1. 点击 **「上传文档 (.pdf/.md/.txt)」**  
2. 选择准备好的 `demo-tech-stack.md` 或 README  
3. 等待提示「上传成功，文档正在入库」  
4. **停顿 15～20 秒**（口播：后台在解析）  
5. 在 **知识库挂载** 列表勾选刚上传的文档  

**检查点：** 文档出现在资源池列表，已勾选挂载。

---

### 镜头 4 · 第一个问题（能答）（1:30～2:30）

**画面：** 中间输入框。

**旁白：**

> 我问一个和文档相关的问题：项目用了哪些技术栈？

**操作：**

1. 输入：`项目用了哪些技术栈？`  
2. 按 Enter 发送  
3. 等待 **流式输出** 完成  

**旁白（回答过程中）：**

> 可以看到回答是流式逐字出现的。下面会列出 Source 引用，对应检索到的文档片段。

**检查点：**

- 回答提到 React、FastAPI、ChromaDB 等  
- 有 `[Source 1]` 或下方来源块  
- **鼠标滚一下** 指出来源区域  

---

### 镜头 5 · 第二个问题（拒答）（2:30～3:10）

**画面：** 仍在同一对话。

**旁白：**

> 我再问一个文档里没有的问题，展示拒答能力。

**操作：**

1. 输入：`明天北京天气怎么样？`  
2. 发送  

**旁白：**

> 系统返回「根据现有资料，我无法回答这个问题」——这是 Prompt 里规定的，资料不足不瞎编。

**检查点：** 出现标准拒答文案。

---

### 镜头 6 · 领域冲突（可选，3:10～3:40）

**若时间紧可跳过。**

**旁白：**

> 如果挂载的文档 tag 和对话不一致，右侧会出现红色领域冲突提醒。

**操作（可选）：**

- 若有第二份不同 tag 的文档可演示勾选；否则 **口播说明即可**，不操作。

---

### 镜头 7 · 架构收尾（3:40～4:20）

**画面：** 可切换到 Cursor 打开 `docs/01-设计思路与架构.md` 架构图，或继续停在前端。

**旁白：**

> 架构上是 React 前端 + FastAPI 后端，SQLite 存对话，ChromaDB 存向量。  
> 问答走 RAG：检索 Top-K 片段，SSE 流式返回。项目已部署在 Render，链接在 README。  
> 以上是 X-TechInsight 的演示，谢谢观看。

**操作：** 可选展示架构图 5 秒，或展示 GitHub / README 一页。

## 三、旁白全文（连贯版，可照着读）

```
这是 X-TechInsight，一个基于 RAG 的技术文档问答平台。用户可以上传 PDF、
Markdown 或文本，系统把文档切分、向量化之后，提问时先检索最相关的片段，
再让大模型基于这些片段回答，并标注来源。

我先新建一个对话，名称叫「技术栈讨论」，领域标签选「后端」。这个标签会在
检索时过滤，只查同领域的文档。

然后我上传一份 Markdown，系统会在后台异步解析入库。等十几秒后，我在右侧
勾选挂载这份文档。

第一个问题：项目用了哪些技术栈？可以看到回答是流式输出的，下面有 Source
引用，对应检索到的文档片段。

第二个问题我问：明天北京天气怎么样？文档里没有这类信息，所以系统明确拒答，
不会编造——这是 RAG 里很重要的幻觉控制。

整体架构是 React 加 FastAPI，SQLite 存对话历史，ChromaDB 存向量，问答接口
用 SSE 流式推送。项目已经部署在 Render 上公网可访问。谢谢观看。
```

## 五、常见问题（录屏时）

| 情况 | 应对 |
|------|------|
| 第一次请求 Failed to fetch | 暂停 30 秒，刷新，从镜头 2 重录 |
| 回答仍是拒答 | 文档可能未入库完成；多等 20 秒重问，或用更直白的问题：「前端用什么框架？」 |
| 后端很慢 | 口播一句「Render 免费版冷启动」；剪辑时可将等待段加速 2x |
| 录错了 | Mac 录屏可在「预览」里简单裁剪头尾 |
